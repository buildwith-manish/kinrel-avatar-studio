# Kinrel Avatar Studio

> A modular, layer-based avatar creation system for the **Kinrel** app.
> This is the standalone V1 scaffold — self-contained, no Kinrel backend
> dependency. Will be merged into the main Kinrel repo later as an
> `avatar_studio` package/module.

## Status

**V1 — base bodies + hair + glasses + accessories populated, local
persistence wired up, CI/CD live.** All 12 standardized base body PNGs
render in the editor; 20 placeholder hairstyle PNGs (10 male + 10
female), 4 glasses styles, and 2 accessories (watch + necklace) are
in place. The Save button persists the avatar locally via
`SharedPreferences` so it survives app restart. GitHub Actions CI runs
`flutter analyze`, `flutter test`, `scripts/check_assets.py --strict`,
and builds a release APK on every push to `main`.

Clothing is **baked into the base body** for V1 — see
[`docs/V1_CLOTHING_DECISION.md`](docs/V1_CLOTHING_DECISION.md) for the
rationale and V2 migration plan.

## What V1 includes

- ✅ `AvatarLayer` enum with 9 layers in the spec-required render order
  (bottom → top): base body, clothing, face-detail (V2), hair, facial
  hair, eyes/eyebrows (V2), earrings, glasses, accessories.
- ✅ `AvatarConfig` immutable data model with `toJson()` / `fromJson()`
  — backend-agnostic, ready to drop into a Supabase `jsonb` column later.
- ✅ `AvatarRenderer` widget — fixed 2:3 aspect, `Positioned.fill` stack,
  graceful fallback to colored placeholder rectangles when an asset is
  missing.
- ✅ `AvatarEditorScreen` — live preview + horizontal pickers per layer
  + Save (persists to local storage), Reset (clears saved state), and
  Copy JSON buttons.
- ✅ 12-body age/gender selector (child → elderly × male/female) with
  real PNG thumbnails in the picker chips.
- ✅ 20 placeholder hairstyles (10 male + 10 female), 4 glasses styles,
  2 accessories — generated by `scripts/generate_placeholder_layers.py`.
  Replace with real artwork any time; the editor picks up new PNGs
  automatically on the next build.
- ✅ Local persistence via `SharedPreferences` (`AvatarStorage`) — saved
  avatar loads on next app start, survives restart, no backend needed.
- ✅ Dynamic asset discovery via `AssetManifest.json` — add a PNG, rebuild,
  it shows up in the picker. No hardcoded asset lists.
- ✅ GitHub Actions CI: `flutter analyze` + `flutter test` +
  `scripts/check_assets.py --strict` + release APK build on every push
  to `main` (artifact downloadable from the Actions run page).
- ✅ GitHub Actions Release: tag a commit `vX.Y.Z` and a GitHub Release
  with the APK attached is auto-published.
- ✅ Tests: `AvatarConfig` round-trip, `AvatarLayer` ordering, base-body
  catalog, `AvatarStorage` (8 cases incl. corrupt-JSON tolerance),
  standardized integration, and placeholder-layer discovery.

## What V1 explicitly does NOT include (per spec)

- ❌ MediaPipe / selfie-based auto-detection (V2).
- ❌ Eyes / eyebrows / nose / mouth swappable layers (V3).
- ❌ Skin-tone tinting / recoloring logic (V1 has a non-functional
  placeholder dropdown).
- ❌ Backend / Supabase integration — local state only.
- ❌ Animations or expressions.

## Getting started

### Prerequisites

- Flutter `>=3.19.0` (Dart `>=3.3.0`)
- Android Studio / VS Code / command-line Flutter tooling

### Run

```bash
# 1. Clone
git clone https://github.com/buildwith-manish/kinrel-avatar-studio.git
cd kinrel-avatar-studio

# 2. Generate platform folders (the scaffold ships without android/ios/
#    web/ folders so the repo stays lean and Flutter-version-agnostic).
flutter create --org com.kinrel --project-name kinrel_avatar_studio .

# 3. Install dependencies
flutter pub get

# 4. Run on a device or emulator
flutter run
```

You should see the editor screen with:

- A 2:3 preview pane showing the saved avatar (or default `adult_male` on
  first launch). A small "Saved locally" badge appears under the preview
  once you've pressed Save at least once.
- Horizontal picker rows for: Base Body (real PNG thumbnails), Clothing
  (V1: only "default" with a "Baked into body" sublabel), Hair, Facial
  Hair (male bodies only), Earrings, Glasses, Accessories.
- A non-functional Skin Tone selector (V1 placeholder).
- A bottom bar with three actions: **Reset to default** (icon, left),
  **Save Avatar** (primary, center), **Copy JSON to clipboard** (icon,
  right). Save persists to `SharedPreferences` so the avatar reloads on
  next app start.

### Test

```bash
flutter test
python3 scripts/check_assets.py --strict
```

### Local persistence

The Save button persists the current `AvatarConfig` to `SharedPreferences`
under the key `kinrel.avatar_config.v1`. On app start, the editor loads
the saved config automatically and shows a small "Saved locally" badge
under the avatar preview. The Reset button (next to Save) clears the
saved state and returns to `AvatarConfig.v1Default()`.

The persistence layer is `AvatarStorage` in `lib/registry/avatar_storage.dart`.
It's the only file that talks to `SharedPreferences` — swapping the
backend to Supabase later means changing only that file plus the call
sites that pass a `userId`.

### CI / CD

GitHub Actions workflows live in [`.github/workflows/`](.github/workflows/):

| Workflow | Trigger | What it does |
| --- | --- | --- |
| [`ci.yml`](.github/workflows/ci.yml) | push to `main`, any PR | `flutter analyze` + `flutter test` + `scripts/check_assets.py --strict` + build release APK + upload as artifact |
| [`release.yml`](.github/workflows/release.yml) | tag push `vX.Y.Z` | build release APK, create a GitHub Release with auto-generated notes, attach the APK |

**Download the APK from a CI run:** go to the **Actions** tab → click the
latest successful `CI` run → scroll to **Artifacts** → download
`kinrel-avatar-studio-apk`.

**Cut a release:**

```bash
git tag v0.2.0
git push origin v0.2.0
# The release.yml workflow will build the APK and publish a GitHub Release.
```

> ⚠️ The CI-built APK is signed with Flutter's auto-generated debug
> keystore (fine for internal distribution / testing). For Play Store
> submission, add a `keystore` secret and configure `android/key.properties`
> in a pre-build step — see
> [Flutter docs](https://docs.flutter.dev/deployment/android#signing-the-app).

## Adding real PNG assets

1. Author / standardize your PNGs against
   [`ANCHOR_SPEC.md`](assets/avatars/ANCHOR_SPEC.md) — every PNG **must**
   be `1024×1536`, RGBA, transparent background, with the documented
   anchor points.

2. Drop them into the right folder using the naming convention from
   `ANCHOR_SPEC.md`. Example:

   ```
   assets/avatars/base/adult_male/body.png
   assets/avatars/hair/male/short_black.png
   assets/avatars/glasses/round_black.png
   ```

3. Re-run `flutter run` (or hot-restart — `AssetManifest.json` is
   regenerated on every build). The new options appear in the picker
   automatically, and the renderer swaps the placeholder rectangle for
   the real PNG.

4. Once a non-base layer folder has its first real PNG, the corresponding
   folder's `.gitkeep` file can be deleted (it's only there to preserve
   empty folder structure in git). Base body folders already have their
   PNGs and no longer carry `.gitkeep`.

## Merging into the main Kinrel app (later)

This scaffold is intentionally self-contained so it can be lifted into
the main Kinrel repo as a path or git submodule. Recommended approach:

1. Move this folder to `<kinrel>/packages/avatar_studio/`.
2. Update the Kinrel app's `pubspec.yaml`:
   ```yaml
   dependencies:
     avatar_studio:
       path: packages/avatar_studio
   ```
3. Replace `lib/main.dart` with a direct import of `AvatarEditorScreen`
   from your app's navigation graph:
   ```dart
   import 'package:avatar_studio/avatar_studio.dart';
   ```
4. Wire `AvatarConfig.toJson()` into your Supabase Person/User table's
   `avatar_config jsonb` column.

The `AvatarConfig` JSON shape is the persistence contract — add new
optional fields only, never rename or remove existing keys.
